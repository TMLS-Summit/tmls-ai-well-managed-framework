# Example Contribution Template

This is an example template that contributors can use when adding content to the framework.

## Adding a New Principle

When adding a new principle to a pillar, use this template:

```markdown
## [Principle Name]

**Description**: Brief, clear explanation of the principle (1-2 sentences)

**Why It Matters**: Executive-level justification - why should leadership care? What risk does it mitigate?

**Implementation Guidance**: Practical steps for technical teams to implement this principle

**Key Indicators**:
- [ ] Measurable criterion 1
- [ ] Measurable criterion 2
- [ ] Measurable criterion 3

**Common Pitfalls**: What mistakes do organizations typically make?

**Related Pillars**: 
- Link to related content in other pillars
- Cross-pillar dependencies

**Example**: Brief real-world example (anonymized if necessary)
```

## Adding a Case Study

When contributing a case study or example, use this template:

```markdown
# Case Study: [Title]

**Industry**: [e.g., Financial Services, Healthcare, Manufacturing]

**Challenge**: What problem was the organization facing?

**Approach**: How did they address it using this pillar's principles?

**Outcome**: What were the results?

**Lessons Learned**: What would they do differently? What worked well?

**Applicable Principles**: List specific principles from this pillar that were applied
```

## Adding Failure Patterns

When documenting anti-patterns or common failures:

```markdown
## ❌ [Failure Pattern Name]

**Description**: What does this anti-pattern look like?

**Why It Happens**: Common reasons organizations fall into this trap

**Consequences**: What are the business and technical impacts?

**Detection**: How can you identify if this is happening in your organization?

**Remediation**: Steps to address and prevent this pattern

**Real-World Impact**: Brief example of consequences (anonymized)
```
